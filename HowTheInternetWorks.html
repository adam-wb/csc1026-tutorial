How the Internet Works

What is the Internet?

The internet is made up of networking infrastructure that lets various devices connect to one another, these devices include servers, PCs and mobiles. This connectivity is based on accepted standards such as the TCP/IP set of protocols.

These protocols specify exactly how devices should communicate. Protocols are responsible for deciding many attributes of a data transmisson including the type, size and structure of any data that is sent or recieved.

Clients and servers are responsible for allowing users to access various services that are found on the internet, including the web. Centralised servers can be used for everyone to connect to, or a peer to peer system can be used.

Clients, Servers & Peers

The client server model is by far the most commonly used in combination with various communication protocols.

Centralised servers are generally favoured over a peer to peer system and are more common.

  • Email clients access your internet service provider's web server to retrieve messages.
  • Your web browser retrieves web pages to be displayed by acessing various web servers where they are stored.

Peer to peer technology on the other hand is used to connect two or more computers and devices. In this case, all devices effectively function as both client and server.

  • Voice over IP or video calling through services like Skype use this peer to peer system.

These include:

  • Web access

  • File transfers
  • Peer to peer data exchanges
  • Instant messaging
  • Shell and console access for computers

TCP/IP

TCP stands for Transmission Control Protocol. It breaks outgoing messages into packets to be sent via IP and reassembles incoming messages. Packets are split into their header and body. The header contains metadata and the body contains the actual data.

IP stands for Internet Protocol. It handles the delivery of packets to a designated IP address, a unique numeric identifier for an internet connected device. For example: 128.246.132.768

The IP protocol routes packets based on the informaton in their header. This routing allows packets to travel through multiple routes and be routed around points of failure. However, there is a risk of data corruption, packets being lost, packets being duplicated or them being delivered in the wrong order.

Because of these risks, IP can only promise best-effort-delivery. To help mitigate these risks, TCP can request that certain packets are retransmitted, re-order the packets it recieves. In addition, if the transmission is taking too long, it can refuse to wait for any more packets and drop the connection.
