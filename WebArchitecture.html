Web Architecture

Web Browsers

Web browsers are software designed to send HTTP requests and process the responses. Many browsers share common layout engines such as Trident, Gecko or Webkit.

Below is a Timeline of Browsers from Wikipedia.


Common Web Browser Features

Feature Notes
Request Generation Assemble URLs and initiates requests.
Response Handling Accept server response codes and act on them.
Content Parsing Read and process HTML and CSS markup.
Maintain State Manage storage of cookies.
Client-side Scripting Run JavaScript supplied by the server.
Encryption Manage SSL transmissions over HTTPS.
Media Objects Display and play media objects embedded in pages.

Web Servers

The use of the term web server refers to software rather than a machine, one PC can run multiple servers. A server will listen on a particular port for HTTP requests and handle them appropriately.

Below is a graph showing the market share of web server companies as of February 2017 from Netcraft.


Common Web Server Features

Feature Notes
Path Translation Locate the resource specified by a URL.
Virtual Hosting Run multiple websites from a single server.
Access Restriction Controls who can access resources.
Log Access and Errors Records events on the server, helping identify errors.
Encrypted Operation Protects data in transit.
Load Balancing Manages traffic and processing resources.
Server Scripting Allows external programs and applications to run.

Key Components of the Web

HTTP covers the format of request and response messages. A URI is a formatted string used to identify a resource. HTML/XHTML is the markup language used to write websites (inlcuding this one!). The DNS or the Domain Name System is the internet's equivalent of a phonebook. TCP/IP is the internet protocol suite covered in more detail here.
