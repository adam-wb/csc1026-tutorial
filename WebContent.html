Web Content

Web Content Overview

Every web document uses HTML to specify content structure and semantics. Other embedded technologies are used to enhance appearance and behaviour of a page. CSS is used for style and layout while scripting languages such as JavaScript (client-side), PHP or Ruby (server side). Source code is delivered to the browser via HTTP or the local file system. This file is then processed by the browser and displayed to the user. The final appearance of a site is a combination of both the author's code and the rendering capabilities of the browser.

It is considered good practice to seperate these different elements of website design. Meaningful structural information such as headings, paragraphs and emphasis should be conveyed in the HTML document. The appearance of a page, such as layout, typefaces and colour schemes should be specified in a CSS file. Any interactive or responsive functionality such as any response to user input or handling various browser inconsitencies should be handled in a seperate scripting file, for example a JavaScript file.

HTML

HTML is made up of tags to specify organisation and structure. For example, headers are specified using <h1> through to <h5> tags. A tag is closed using a forward slash, e.g. </h1>. In more modern versions of HTML such as HTML5 or XHTML, some tags can be self-closing such as the break tag, <br/>. This is considered best practice over writing something like <br></br>. CSS

CSS allows layout and other effects to be applied to given HTML tags or locations in a HTML document. For example, to make all <h1> tags be aligned to the center:

h1 {    
    text-align: center;
}

Style rules can be limited to certain parts of a document wrapped in named <div> and <span> tags. This style rule will only affect elements wrapped inside <div id="picture">, id="picture" is an example of a HTML attribute.

#picture {
          position: relative;
          width: 300px;
          height: 250px;
}

It is important to note that div and span ids cannot start with a number.

JavaScript

JavaScript is a scripting language that executes on the client machine in a web browser. It is embedded into XHTML and arrives wit the webpage. JavaScript uses a syntax very similar to C. There are two ways to add Javascript to a page, one is using inline <style> tags.

<script type="text/javascript">
    window.alert("You pressed the button!");
</script>

Alternatively, JavaScript can be kept in an external file. This allows repeated code to be reused across multiple documents and maintains a good degree of seperation.

<script type="text/javascript"
 src="js/examplescript.js"></script>

The two methods can't be combined, that is, you cannot have a hybrid internal/external <script> block. <script> also cannot be self closed with <script/>

Script Access & The DOM

A Document Object Model defines objects in a document, the relationships between them as well as their methods and properties. The HMTL DOM describes a HTML document and the browser that it is contained in. It allows access to any part of a document according to the hierarchy. Different DOMs are available, but the W3C DOM is considered standard by most browsers.

The DOM can be used to access objects on a webpage.

document.getElementByID("TextBox").value = "Hello";

document.getElementsByTagName("h1");

The DOM can also be used for event handlers, this can be done in multiple ways:

Inline (Found in HTML, not a script file)

<a href="index.html" onclick="function();">Click for script</a>

Within a script:

Traditional

document.getElementById("click").onClick = youClicked

Internet Explorer Compatability (Event attachement)

document.getElementById("click").attachEvent('onClick',youClicked)

W3C Method

document.getElementById("click").addEventListener('click',youClicked,false)

Extensions & Frameworks

Making a good looking website and coding good javascript can be hard work and time consuming, fortunately, there are some tools available to make life easier. This website, for example, is built using Bootstrap, as well as jQuery.

Bootstrap

Bootstrap is an open source framework for HTML, CSS and JavaScript. It is repsonsive, which means it can be used to scale to any sized screen on a desktop, tablet or mobile device. It comes with an assortment of pre-packaged css rules for various html elements in addition to a variety of jQuery plugins. Bootstrap also includes an assortment of icons such as the one used on this website's navigation bar as part of the home button. These are provided by Gylphicon.

jQuery

jQuery is mandatory for various Bootstrap plugins to work, such as the hover over tooltips and pop overs seen on this page just above here. Apart from being necessary for Bootstrap plugins to work, jQuery is useful for writing JavaScript in general. The basic idea behind it is to simplify common functions so they take less time to write. In JavaScript, the start of a jQuery function is denoted by a $. This JavaScript library works across multiple browsers and can find elements through CSS selectors or manipulate CSS rules.
